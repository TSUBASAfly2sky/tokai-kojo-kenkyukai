# 公開手順マニュアル（独自ドメイン版）

このマニュアルは「東海古城研究会」のホームページを **GitHub Pages + 独自ドメイン `tokai-kojo-kenkyukai.jp`** で公開するための完全手順書です。
プログラミング初心者の方でも順番にやれば必ず公開できるよう、画面の動きまで細かく書いてあります。

所要時間：**約30分〜1時間**（DNS反映待ちの時間を除く）

---

## 全体の流れ

1. 【パート1】GitHubアカウントを準備（5分）
2. 【パート2】リポジトリを作ってファイルをアップロード（10分）
3. 【パート3】GitHub Pages を有効化して仮URLで表示確認（5分）
4. 【パート4】GitHub側で独自ドメインを設定（3分）
5. 【パート5】DNS設定（ドメイン管理画面）でAレコード追加（10分）
6. 【パート6】反映待ち＆HTTPS有効化（数十分〜最大24時間）

---

## 【パート1】GitHubアカウントを準備

### 1-1. アカウント作成（既にお持ちならスキップ）

1. https://github.com/signup にアクセス
2. メールアドレス・パスワード・ユーザー名を入力して登録
3. 確認メールのリンクをクリックして本登録完了

> 💡 **ユーザー名はあとで使うのでメモしておきましょう**（例：`tsubasa-okada`）

---

## 【パート2】リポジトリ作成 & ファイルアップロード

### 2-1. リポジトリ作成

1. GitHubにログインした状態で、画面右上の **「+」マーク** → **「New repository」** をクリック
2. 入力項目：
   - **Repository name** … `tokai-kojo-kenkyukai`（半角英数とハイフンのみ）
   - **Description**（任意）… `東海古城研究会 公式サイト`
   - **Public** を選択（無料プランで GitHub Pages を使うには Public が必要）
   - 「Add a README file」のチェックは **入れない**（後でアップロードするため）
3. 緑の **「Create repository」** ボタンをクリック

### 2-2. ファイルをアップロード

1. 作成直後の画面で **「uploading an existing file」** という青いリンクをクリック
2. 配布ZIP `tokai-kojo-site.zip` を **解凍して中身を確認**
3. 解凍されたフォルダの中身（**フォルダごとではなく、中のファイル＆フォルダ全部**）を、ブラウザ画面のグレーのエリアに **ドラッグ&ドロップ**
   - アップロード対象：
     - `index.html`
     - `report.html`
     - `admin.html`
     - `data.json`
     - `css/`（フォルダごと）
     - `js/`（フォルダごと）
     - `.nojekyll`（隠しファイル）
     - `CNAME`
     - `README.md`
4. 全部アップロードされたら、画面下部 **「Commit changes」** ボタンをクリック

> ⚠️ **`.nojekyll` ファイルが表示されない場合**：
> Macなら Finder で **`Cmd + Shift + .`**（隠しファイル表示）、Windowsなら エクスプローラの「表示」→「隠しファイル」にチェックを入れて、`.nojekyll` も忘れずにアップロードしてください。

---

## 【パート3】GitHub Pages を有効化（仮URLで表示確認）

### 3-1. Pages設定を開く

1. リポジトリのトップ画面で、上のメニューバーから **「Settings」** をクリック
2. 左サイドバーの **「Pages」** をクリック

### 3-2. 公開設定

1. **「Source」** が「Deploy from a branch」になっていることを確認
2. **「Branch」** で **「main」** を選択、フォルダは **「/ (root)」** のまま
3. **「Save」** をクリック

### 3-3. 公開確認

1. 1〜2分待ってからページを再読み込み
2. 上のほうに「Your site is live at https://（ユーザー名）.github.io/tokai-kojo-kenkyukai/」と緑色のメッセージが出れば成功
3. リンクをクリックして、サイトがちゃんと表示されるか確認

> ⏰ **すぐに表示されない場合**：5分ほど待ってリトライしてください。

---

## 【パート4】独自ドメインを GitHub に設定

### 4-1. カスタムドメインを入力

1. 同じ「Pages」設定画面の **「Custom domain」** 欄に
   ```
   tokai-kojo-kenkyukai.jp
   ```
   と入力（`https://` や末尾の `/` は不要）
2. **「Save」** をクリック

> 💡 既にリポジトリに `CNAME` ファイルを置いてあるので、自動的にこの欄が埋まっているかもしれません。その場合はそのままでOK。

3. 直後にDNSチェックが走ります。この時点ではまだDNS設定をしていないので **「DNS check failed」とエラーが出ますが正常です**。次のパートでDNS設定をします。

---

## 【パート5】DNS設定（ドメイン取得サイトでの作業）

`tokai-kojo-kenkyukai.jp` を取得したドメイン管理サービス（お名前.com、Xserverドメイン、ムームードメイン、Cloudflare 等）の管理画面にログインします。

### 5-1. 設定する内容

ドメインのDNS設定画面で、**次の5つのレコードを追加**してください：

#### Aレコード（4つ） — Apexドメイン用

| ホスト名 | タイプ | 値 | TTL |
|---|---|---|---|
| `@`（または空欄） | A | `185.199.108.153` | 3600 |
| `@`（または空欄） | A | `185.199.109.153` | 3600 |
| `@`（または空欄） | A | `185.199.110.153` | 3600 |
| `@`（または空欄） | A | `185.199.111.153` | 3600 |

> 「ホスト名」を空欄や `@` にすると、`tokai-kojo-kenkyukai.jp` 自体を指します。

#### CNAMEレコード（1つ） — wwwサブドメイン用（推奨）

| ホスト名 | タイプ | 値 | TTL |
|---|---|---|---|
| `www` | CNAME | `（あなたのGitHubユーザー名）.github.io.` | 3600 |

> 値の末尾の **ピリオド「.」** を忘れないでください（必須）。
> 例：ユーザー名が `tsubasa-okada` の場合は `tsubasa-okada.github.io.`

### 5-2. 主要サービス別の設定画面の場所

<details>
<summary>📂 お名前.com の場合</summary>

1. ログイン後「ドメイン」→「ドメイン機能一覧」
2. 「DNS関連機能の設定」→ ドメイン選択 → 「DNSレコード設定を利用する」
3. レコード追加画面で上記5レコードを1件ずつ追加
4. 最後に「確認画面へ進む」→「設定する」
</details>

<details>
<summary>📂 Xserverドメイン / エックスサーバーの場合</summary>

1. サーバーパネル → 「DNSレコード設定」
2. 対象ドメインを選択 → 「DNSレコード追加」タブ
3. 上記5レコードを追加
</details>

<details>
<summary>📂 ムームードメインの場合</summary>

1. コントロールパネル → 「ドメイン管理」→「ムームーDNS」
2. 対象ドメインの「変更」
3. 「カスタム設定」を選択して上記レコードを追加
</details>

<details>
<summary>📂 Cloudflare の場合</summary>

1. ダッシュボード → 対象ドメイン → 「DNS」
2. 「Add record」で上記5レコード追加
3. **重要**：Aレコード4つの「Proxy status」は **「DNS only」（グレーの雲）** にしてください。オレンジ雲のままだとGitHub Pagesでは動きません。
</details>

<details>
<summary>📂 Value-Domain の場合</summary>

1. コントロールパネル → 「ネームサーバー設定」→「DNSレコード/URL転送の設定」
2. テキストエリアに以下を入力：
   ```
   a @ 185.199.108.153
   a @ 185.199.109.153
   a @ 185.199.110.153
   a @ 185.199.111.153
   cname www （ユーザー名）.github.io.
   ```
3. 「保存」
</details>

## 【パート6】反映待ち＆HTTPS有効化

### 6-1. DNS反映を待つ

DNSの変更は世界中に伝わるのに **数十分〜最大24時間** かかります。
通常は1〜2時間程度で反映されます。

### 6-2. 反映確認

ブラウザで `https://tokai-kojo-kenkyukai.jp` にアクセスして、サイトが表示されるかチェック。

または、コマンドラインで確認することもできます（できる方のみ）：

```bash
# Macのターミナル or Windowsのコマンドプロンプトで
nslookup tokai-kojo-kenkyukai.jp
```

`185.199.108.153` などのIPが返ってくれば成功です。

### 6-3. HTTPSを有効化

DNS反映後（サイトが独自ドメインで表示できるようになってから）：

1. GitHub のリポジトリ → Settings → Pages を開く
2. ページ下部 **「Enforce HTTPS」** にチェックを入れる
   - チェックボックスがグレーアウトしている場合は、まだSSL証明書が発行されていません。30分〜数時間待ってからリトライ
3. これで `https://` で安全にアクセスできるようになります

> 🎉 完了です！

## 【任意】更新・運用のヒント

### 内容を更新するには

1. `admin.html` にアクセス（例：`https://tokai-kojo-kenkyukai.jp/admin.html`）
2. パスワード `tokai1960` でログイン（初回は必ず変更を）
3. お知らせや活動報告を編集
4. 右上「📥 JSON書き出し」で `data.json` をダウンロード
5. GitHubのリポジトリで `data.json` を開き、編集ボタン（鉛筆マーク）
6. 中身を全部消して、ダウンロードした `data.json` の中身を貼り付け
7. 「Commit changes」をクリック
8. 1〜2分後、本番サイトに反映されます

### admin.html を直接見られないようにしたい場合

`admin.html` はパスワード保護されていますが、URLを知っていれば誰でもログイン画面にはアクセスできます。
URLを推測されないように、ファイル名を変更することもできます：

1. `admin.html` の名前を例えば `kanri-okada-2026.html` のように推測されにくい名前に変更
2. `index.html` と `report.html` のフッターにある「管理画面」リンクも合わせて変更
3. GitHubで上書きコミット

## 困ったときは

### サイトが表示されない
- `Settings → Pages` で「Your site is live at...」が表示されているか確認
- ブラウザのキャッシュをクリアして再アクセス
- 数分待ってから再度試す

### 独自ドメインで表示されない
- DNS反映に時間がかかっている可能性（最大24時間）
- DNS設定が正しいか https://www.whatsmydns.net/ でチェック（`tokai-kojo-kenkyukai.jp` を A レコードで検索 → 結果が `185.199.108.153` などになっていればOK）

### HTTPS のチェックボックスがグレーアウトしている
- まだSSL証明書が発行されていません。30分〜数時間待ってからリトライしてください

### お問い合わせフォームが表示されない
- ブラウザの設定でiframeがブロックされている可能性。シークレットモード等で確認
